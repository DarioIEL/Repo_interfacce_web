# Il modificatore `static` in Java

## Cos'è

`static` indica che un membro appartiene alla **classe**, non alle istanze. Esiste una sola copia in memoria, indipendentemente da quante istanze vengono create.

---

## 1. Variabili statiche (class variables)

Condivise tra tutte le istanze. Utili per contatori, configurazioni globali, valori comuni.

```java
public class Contatore {
    static int totale = 0;
    String nome;

    public Contatore(String nome) {
        this.nome = nome;
        totale++;
    }
}

Contatore a = new Contatore("Alice");
Contatore b = new Contatore("Bob");
Contatore c = new Contatore("Carlo");

System.out.println(Contatore.totale); // 3
```

Ogni istanza incrementa la **stessa** variabile `totale`. Se fosse non-static, ogni istanza avrebbe la propria copia e il risultato sarebbe sempre `1`.

**Accesso corretto:** tramite il nome della classe, non tramite un'istanza.

```java
// ✅ Corretto
System.out.println(Contatore.totale);

// ⚠️ Funziona ma è fuorviante — evitare
System.out.println(a.totale);
```

---

## 2. Metodi statici

Non richiedono un'istanza per essere chiamati. Tipici nelle classi di utilità.

```java
public class StringUtils {
    public static String inverti(String s) {
        return new StringBuilder(s).reverse().toString();
    }

    public static boolean isPalindromo(String s) {
        return s.equals(inverti(s));
    }
}

System.out.println(StringUtils.inverti("java"));       // "avaj"
System.out.println(StringUtils.isPalindromo("radar")); // true
System.out.println(StringUtils.isPalindromo("hello")); // false
```

**Vincolo critico:** un metodo statico non può accedere a variabili o metodi d'istanza. Non esiste `this` nel contesto statico.

```java
public class Esempio {
    int valore = 10;

    public static void stampa() {
        System.out.println(valore); // ERRORE di compilazione
        // Non static field 'valore' cannot be referenced from a static context
    }
}
```

Per usare stato d'istanza dentro un metodo statico, bisogna passare l'oggetto come parametro:

```java
public class Esempio {
    int valore = 10;

    public static void stampa(Esempio e) {
        System.out.println(e.valore); // ✅ OK
    }
}
```

## 3. Costanti — il pattern `static final`

Il modo idiomatico per definire costanti in Java. `static` perché appartiene alla classe, `final` perché non può essere riassegnata.

```java
public class Fisica {
    public static final double VELOCITA_LUCE = 299_792_458.0; // m/s
    public static final double GRAVITA       = 9.81;          // m/s²
    public static final double PI            = Math.PI;
}

double energia = 5.0 * Math.pow(Fisica.VELOCITA_LUCE, 2);
System.out.println("E = " + energia);
```

Convenzione: costanti in `UPPER_SNAKE_CASE`.

## Riepilogo

| Tipo | Appartiene a | Una sola copia | Accesso senza istanza |
|---|---|---|---|
| Variabile static | Classe | ✅ | ✅ |
| Variabile d'istanza | Oggetto | ❌ | ❌ |
| Metodo static | Classe | — | ✅ |
| Metodo d'istanza | Oggetto | — | ❌ |

## Trappola comune

L'abuso di variabili statiche mutabili introduce **stato globale implicito** — una delle cause più frequenti di bug difficili da tracciare, specialmente in contesti multi-thread dove più thread modificano la stessa variabile senza sincronizzazione.

```java
public class Cache {
    static List<String> dati = new ArrayList<>(); // ⚠️ stato globale mutabile

    public static void aggiungi(String s) {
        dati.add(s); // non thread-safe
    }
}
```

Se necessario usare stato statico condiviso, valutare `synchronized` o strutture thread-safe come `ConcurrentHashMap`.
